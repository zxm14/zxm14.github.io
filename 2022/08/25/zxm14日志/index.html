2022年08月25日
约408字 预计需要1分钟

由于没有系统地架构起来hexo博客的知识框架，忘记了博客的维护方法，我删除了第一次制作的hexo博客，并重新配置了一个。但在二次配置的过程中涌现了很多问题：

2022.8.25

  1. CMD无法识别中文：修改注册表是字符集为utf-8，但还是一直失败，于是参考其他人换了个字体。比较坑的是网上教程中的字体不管用，我挨个尝试，最终换了另外一种字体。（参考）
  2. 主题上传失败：切换了npm镜像源，并移动文件保存重启（注意cmd的路径一定要对）、
  3. 出现npm WARN config global --global, --local are deprecated. Use --location=global instead，更新版本后无改善，又不想安装PowerShell，于是准备修改文件。但是没有找到名为npm.cmd的文件。后来发现是一个应用程序，应该右键编辑而不是用什么方法打开，除非你想运行它。（参考）
  4. 开始hexo d的时候又出现Please make sure you have the correct access rights and the repository exists，我第一次配的时候没有这种情况，可恶！不过按照这个教程成功解决。
  5. 然后又研究nexmoe主题的种种，就是看文档和文件解决的，没有大问题。但出现了封面图片无法显示的bug
  6. （总结）找一个合适的主题好累，整理这种工程向的项目也好累，等以后我前端技术好点一定要自己写一个适合我的主题。虽然大概率还是要咕咕
